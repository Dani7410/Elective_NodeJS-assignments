How to set up app.js

App.js contains the website event handling file. If configures all our routes on the website. In the first line of the pictures, you see a variable instantiating express. It gives us the option to use express modules (place it in the top of the file). On the second line we see a new variable called “app”. This is how we are going to use the modules. And then on third we are telling app should also include our “public” directory which consist of html pages and other fun stuff. Now we have made it possible to start routing. Except we need to give it a port to serve the website.

Callback Function

Title description, Sep 2, 2017

Notice the arrowheads. That is called a callback Function. A Callback is a function passed as an argument to another function which then uses it. What it basically does it that it allows for the program to keep running the next thing it needs to do, until the function in the callback is done processing. And then call back to that function that is processing when its ready to be used.

in other terms also called an Asynchronous process. (unblocking.)

is asynchronous by default. That means that the app runs on a single thread processing multiple instructions. Kind of like a waiter at a restaurant. The waiter takes and order, deliver the order to the kitchen and then proceeds to take orders. When the food is done the waiters goes back to the kitchen, take the food, and serves it.
